💀 NFC RING USER GUIDE 💀

📌 OVERVIEW

NFC Ring is a Progressive Web App (PWA) designed for EMV card skimming, cloning, and testing using WebNFC. It works completely offline and supports custom APDU commands, preset commands, and auto-fill for payment forms.

⚡ REQUIREMENTS

  • Device: Android 10+ (Chrome 89+) or iOS 15+ (Safari, limited functionality)
  • Browser: Chrome (recommended) or Safari
  • NFC Tags: NTAG215 or NTAG216 (for cloning)
  • EMV Cards: Contactless Visa/Mastercard (for skimming)
  • Internet: Only required for initial load (works offline after)

🚀 GETTING STARTED

1️⃣ Install the App

Open the app in Chrome on Android or Safari on iOS. The browser will prompt you to install the PWA. Accept the prompt to add it to your home screen. No install button is visible—it uses the browser's native PWA prompt.

2️⃣ Enable NFC

Ensure NFC is enabled on your device:

  • Android: Settings > Connected Devices > NFC
  • iOS: Settings > NFC (limited to NDEF tags)
For full EMV functionality, use Chrome on Android.

3️⃣ Enable Chrome Flags (Android Only)

For raw EMV reading, enable WebNFC in Chrome:

  1. Open Chrome and go to: chrome://flags/#enable-experimental-web-platform-features
  2. Enable the flag: Experimental Web Platform features
  3. Restart Chrome

🔧 COMMANDS

The app uses a terminal-style interface. Type the following commands in the input field:

Command Description
help Show this help message
scan Scan an EMV card (hold card near device)
clone Clone the last scanned card to an NTAG215/216 tag
test Test a cloned tag (check if it works as a payment card)
autofill on Enable auto-fill for payment forms
autofill off Disable auto-fill
list List all saved cards
clear Clear the terminal log
generate Generate a test card for encoding
manual Manually enter card data

💳 SKIMMING EMV CARDS

To skim an EMV card:

  1. Type scan and press Enter.
  2. Hold the contactless EMV card near the back of your device.
  3. The app will read the card data and display it in the terminal.
  4. The card data is automatically saved for later use.

> scan [NFC] Scanning for EMV card... [NFC] Tag detected (UID: A1 B2 C3 D4) [NFC] EMV data read successfully! [DATA] PAN: 411111******1111 [DATA] Expiry: 12/25 [DATA] UID: A1B2C3D4 [STORAGE] Card saved successfully!

🔄 CLONING CARDS TO NTAG215/216

To clone a card to an NFC tag:

  1. Ensure you have a blank NTAG215 or NTAG216 tag (NTAG213 will NOT work).
  2. Type clone and press Enter.
  3. Hold the NTAG215/216 tag near the device.
  4. The app will write the EMV data to the tag.
  5. Once complete, the tag can be used for tap-to-pay (if the terminal accepts it).

⚠️ IMPORTANT:
  • NTAG213 does NOT support ISO 14443-4 Type A emulation (required for EMV). Use NTAG215 or NTAG216.
  • Some POS terminals may reject cloned tags due to security checks (e.g., dynamic CVC, transaction counters).
  • Test on older terminals or offline POS systems for best results.

🧪 TESTING CLONED TAGS

To test if a cloned tag works:

  1. Type test and press Enter.
  2. Hold the cloned NTAG215/216 tag near the device.
  3. The app will verify the tag's data and display the results.

> test [NFC] Hold cloned tag near device... [NFC] Tag detected (UID: A1B2C3D4) [TEST] PAN: 411111******1111 ✅ [TEST] Expiry: 12/25 ✅ [TEST] Track 2 Data: Present ✅ [TEST] Result: TAG IS PAYMENT-READY!

🔧 TROUBLESHOOTING CLONING ISSUES

❌ Tag Not Detected

Possible Causes:

  • Tag is not NTAG215/216 (NTAG213 will not work).
  • Tag is locked (write-protected).
  • Tag is too far from the device.
Solutions:
  • Use a new NTAG215/216 tag.
  • Check if the tag is rewritable (some tags are locked by default).
  • Hold the tag closer to the device.

❌ Cloning Fails

Possible Causes:

  • Tag is not blank (contains old data).
  • Tag is damaged.
  • NFC is disabled on the device.
Solutions:
  • Use a fresh NTAG215/216 tag.
  • Enable NFC on your device.
  • Restart the app and try again.

❌ Tap-to-Pay Not Working

Possible Causes:

  • POS terminal rejects cloned tags (security checks).
  • Tag is not properly encoded.
  • Missing required EMV data (e.g., ATC, cryptograms).
Solutions:
  • Try a different POS terminal (older terminals work better).
  • Use the test command to verify the tag data.
  • For better compatibility, use the APDU tool to manually adjust EMV data.

📝 MANUAL CARD ENTRY

If you don't have a physical card to scan, you can manually enter card data:

  1. Type manual and press Enter.
  2. Fill in the card details (PAN, expiry, etc.).
  3. The app will auto-generate missing fields (e.g., UID, ATC).
  4. Save the card and clone it to a tag.

🎯 GENERATING TEST CARDS

The app includes predefined test cards for testing:

  1. Type generate and press Enter.
  2. Select a test card (e.g., Visa Test Card, Mastercard Test Card).
  3. The app will generate a valid test card that can be cloned to a tag.

Amex Test Card (378282246310005) [GENERATE] Enter choice (1-3): 1 [GENERATE] Test card generated (PAN: 4111111111111111, Expiry: 12/25) [STORAGE] Card saved successfully!

🔐 AUTO-FILL FOR PAYMENT FORMS

Enable auto-fill to automatically fill payment forms with cloned card data:

  1. Type autofill on and press Enter.
  2. Open a payment form (e.g., Stripe test page).
  3. Focus on a card number field—the app will auto-fill the details.

⚠️ NOTE:
  • Auto-fill works best on unprotected payment forms.
  • Some websites may block auto-fill for security reasons.

🛠️ ADVANCED APDU COMMANDS

For advanced users, the APDU tool allows sending custom commands to NFC tags:

  1. Go to the APDU tab.
  2. Enter a custom APDU command (e.g., 00A404000E325041592E5359532E4444463031).
  3. Hold an NFC tag near the device.
  4. View the response and interpretation.

80CA9F3600 Read ATC (Application Transaction Counter) ATC (e.g., 00 10 90 00)

APDU Response Interpretation

APDU responses include status words (SW1/SW2) that indicate success or failure:

SW1/SW2 Meaning
90 00 Success - Command executed successfully.
6A 82 File not found - The requested file or data object does not exist.
6A 81 Function not supported - The command is not supported by the tag.
6A 86 Incorrect P1/P2 - Parameters are invalid.
69 85 Conditions not satisfied - Tag is locked or in an invalid state.
63 00 Verification failed - Authentication or verification failed.
